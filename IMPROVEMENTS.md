# Reston — список улучшений

## Влияет ли что-то на работу сайта прямо сейчас?

**Нет.** Все пункты ниже — это технический долг и риски, но сайт работает стабильно.
Единственный пограничный случай — отсутствие retry при IIKO: если IIKO API временно недоступен,
синхронизация тихо падает. Но это не ломает UI — пользователь просто видит старые данные.

## Приоритет 1 — Чистый код (начинаем здесь)

| # | Файл | Проблема | Что делать |
|---|------|----------|------------|
| 1 | `frontend/src/pages/AdminPage.tsx` | 118 KB в одном файле, ~2500 строк | Разбить на под-компоненты по вкладкам |
| 2 | `backend/app/api/recipes.py` | 56 KB, всё в одном файле | Вынести бизнес-логику в сервисный слой |
| 3 | `backend/app/api/admin.py` | 47 KB, смешана логика роутов и бизнес-правила | Аналогично — сервисы отдельно |
| 4 | Дублирование типов | Frontend типы пишутся вручную, дублируя Pydantic схемы | Генерация типов из OpenAPI |

## Приоритет 2 — Надёжность

| # | Проблема | Риск | Решение |
|---|----------|------|---------|
| 5 | Нет retry при IIKO сбоях | Тихие ошибки синхронизации | Tenacity / exponential backoff |
| 6 | Нет тестов | Регрессии при изменениях | Pytest для критичных эндпоинтов |
| 7 | Один PostgreSQL | Потеря данных при сбое сервера | Read-реплика или бэкап-стратегия (уже есть daily backup) |

## Приоритет 3 — Безопасность

| # | Проблема | Риск | Решение |
|---|----------|------|---------|
| 8 | CORS разрешает localhost в проде | Низкий, только dev | Убрать localhost из production CORS |
| 9 | Нет rate limiting на API | DDoS / brute-force на /auth/login | SlowAPI middleware |
| 10 | Нет request-signing для внешних сервисов | Теоретический | HMAC подпись при вызовах Telegram/Google |
